# 更好的外墙处理使用说明书

## 功能概述

这个插件用于在 OrcaSlicer 导出 G-code 时对外墙进行二次处理，目标是消除外墙接缝痕迹。

当前版本（v0.4.0）核心功能是**亚层螺旋（旋转花瓶式接缝消除）**，另保留立体斜接缝与平面渐变两种回退模式，以及二次整形。

它是导出阶段的后处理插件，不参与 OrcaSlicer 内部切片规划，因此不会修改内部路径预览和内部打印时间估计。

## 亚层螺旋原理

普通打印每层外墙在同一高度闭合一圈，起终点处必然留下接缝。亚层螺旋把整层外墙改写成连续螺旋：

- 从前一层高度出发，Z 沿路径弧长匀速爬升，每圈升高 `层高 / x`，转 x 圈到达本层高度（x 即超分倍数）。
- 每圈流量为主体流量的 1/x，与每圈 `层高/x` 的沉积厚度精确匹配。
- 首圈流量按间隙线性渐升（下方是平的前层顶面，间隙从 0 长到 层高/x）。
- 到顶后沿轮廓补一圈**互补流量**（从满流量线性降到零），填平螺旋顶面遗留的楔形。
- 最后可选**零流量空转一圈**，把顶面压平整，为下一层提供平坦基面。
- **亚层级精度优化（XY 插值）**：子圈不垂直堆叠——把本层轮廓视为层片中点的表面采样，
  子圈在高度分数 t 处把 XY 向相邻层轮廓插值（下半程向前一层渐变、上半程向下一层，
  端点权重 0.5），用相邻两层加本层共三层的状态重建表面斜率。斜面上的一级大台阶
  被摊成 x 级小台阶，垂直细节真实提升。

除首尾外，中间各圈是纯连续螺旋——**完全没有接缝**，Z 与流量全程连续，无任何压力突变点。总挤出量精确等于 主体流量 × 原外墙挤出量。

## 三种接缝模式

- `spiral 亚层螺旋`：如上，效果最好。要求外墙是闭合回路且层高可知，不满足时该回路自动退化为 scarf。
- `scarf 立体斜接缝`：只在起始斜拼长度内 Z 爬升，结尾补互补重叠段。适合闭合性差的模型。
- `flat 平面流量渐变`：Z 不动，接缝两端做流量渐变。最保守。

首层（无法确定层高）自动按 flat 处理，避免 Z 下探撞热床。最顶层（无下一层轮廓）螺旋照常：插值把下半程拉伸到整层——从上一层螺旋的收尾位置（与前层的中点轮廓）出发，沿整层匀速渐变到自身轮廓，与上一层连续衔接；scarf 模式的顶层仍退化为 flat。

## 兼容性

- 支持绝对挤出（M82）与相对挤出（M83），并正确处理 `G92 E` 重置。
- G2/G3 圆弧（IJ 圆心格式）按采样精度展开为直线段后正常处理；仅 R 格式或半径不一致的异常圆弧回路跳过并报告数量。
- 外墙中出现回抽或超过 0.5mm 的空移时，回路在该处截断（截断后的开环不做螺旋，退化为 scarf/flat）。
- 插入段结束后自动恢复原始进给速度；若外墙结束后已回抽，插件会在插入段开始前回填、结束后重新回抽相同长度。
- 已处理过的文件（文件头带 `BOWP` 标记）再次处理时自动跳过，保证幂等。

## OrcaSlicer 后处理用法

1. 用 `launch_settings.bat` 打开独立中文设置器
2. 在 OrcaSlicer 中进入 `Process Settings > Others > Post-Processing Scripts`
3. 把 `orca_postprocess_command.txt` 里的命令整行粘进去
4. 将 `<INSTALL_DIR>` 替换为项目的实际安装目录
5. 之后每次导出 G-code 时，Orca 都会自动调用脚本

## 参数说明

### 接缝处理

- `接缝模式`（`--seam-mode spiral|scarf|flat`）
  见上文三种模式。默认 `spiral`。

- `超分倍数(圈/层)`（`--seam-superres`，默认 `4`）
  螺旋模式：每层转几圈（每圈层高 = 层高/x，流量 1/x）。受打印机最小可靠挤出层厚限制。
  斜接缝模式：斜拼区渐变细分段数。

- `动态超分倍率`（`--enable-dynamic-superres` / `--disable-dynamic-superres`，默认开）与
  `子圈重叠比例(线宽)`（`--spiral-overlap`，默认 `0.5`）、
  `动态超分上限(圈)`（`--superres-max`，默认 `12`）
  平坦/缓坡处层间轮廓横向偏移大，固定倍数的子圈横向错开量可能超过
  线宽，圈与圈盖不住会留下未覆盖条带（上下层衔接断开）。开启后按
  回路实际使用的插值偏移场（钳制+平滑后）的**最大值**与线宽（由原始
  挤出量反推）自动把倍数提升到相邻子圈至少重叠该比例的线宽，上限
  由"动态超分上限"控制。倍率极高时每圈流量趋近于零，效果自然退化为
  顶面熨烫（贴着表面低流量反复抹平），无害。基础倍数够用时不改变；
  只提升不降低。

- `亚层级精度优化(XY插值)`（`--enable-xy-interp` / `--disable-xy-interp`，默认开）
  见上文原理。关闭后子圈垂直堆叠（只消接缝，不提升斜面细节）。
  层间轮廓差异过大（如文字、横孔边缘）处插值可能产生模型上不存在的斜坡。

- `移除覆盖区原生熨烫`（`--enable-ironing-trim` / `--disable-ironing-trim`，默认开）
  这是**兼容旧方案**：当 G-code 中已经有 Orca 的 `;TYPE:Ironing` 原生熨烫时，
  插件不会再叠加脚本熨烫，而是按本开关修剪会压坏螺旋斜面的原生熨烫。Orca
  熨烫若骑在螺旋建出的斜面环带上，会把斜面重新压平（喷嘴热量本身就足以
  损伤斜面）。开启后按**笔画**为单位整条删除主要（弧长占比 >50%）落在
  覆盖带内的熨烫路径——连同引导空移一起删，不留空走。覆盖带使用螺旋实际
  到达场判定：只有本层上行螺旋或下一层底圈真实覆盖的位置才删除；最顶层、
  大平面内部以及螺旋未到达的轮廓失配区域都会保留原生熨烫。删除数量在总结
  中报告。

- `脚本熨烫裸露顶面`（`--enable-script-ironing` / `--disable-script-ironing`，默认开）
  **推荐使用的新方案，可替代切片器熨烫；不要与 Orca 原生熨烫同时开启。**
  当导出的 G-code 中没有 `;TYPE:Ironing` 时，插件会在后处理阶段自动对裸露顶面
  生成平行直线小流量熨烫：采样点必须在当前外墙轮廓内，同时不被下一层轮廓
  覆盖，也不在本层螺旋实际到达范围内。这样顶层、无法继续收敛的平面区域会
  被脚本熨烫；侧面斜坡和螺旋已经覆盖的区域不会被压平。若文件中检测到 Orca
  原生熨烫，本功能自动不叠加，改用上面的"移除覆盖区原生熨烫"方案。

- `脚本熨烫流量`（`--script-ironing-flow`，默认 `0.10`）与
  `脚本熨烫速度(mm/s)`（`--script-ironing-speed`，默认 `30`）、
  `路径粗细(mm)`（`--path-width`，默认 `0.42`）
  脚本熨烫的挤出比例、移动速度和路径宽度。行距复用"子圈重叠比例"：
  `行距 = 路径粗细 × (1 - 子圈重叠比例)`，默认 0.42mm 路径、0.5 重叠即
  约 0.21mm 行距。

- `接缝补偿长度(mm)`（`--seam-comp-length`，默认 `2.0`）与
  `接缝补偿强度`（`--seam-comp-scale`，默认 `1.0`）
  起步腔压偏高时头几毫米容易挤出凸起，凸起耗掉压力后又紧跟一段欠压
  凹陷。补偿采用**再分配式**：开头补偿长度内按整圈满流量为基准、
  前置加重的二次曲线减料压掉凸起（首圈流量下限钳到零），减掉的料
  **原量摊回**随后 2 倍补偿长度的回补段（权重线性递减）填平欠压凹陷，
  总挤出量精确守恒——加大强度不再造成接缝后净凹陷。
  默认 `1.0`；凸起仍可见调大（1.3~2.0，零流量区间变长），起点本身
  出现凹坑调小（0.4~0.8）；`0` 关闭。另外回填（unretract）已改为摊进
  起步头 2mm 边走边完成，从源头减少静止淤积。

- `排压圈回抽量(mm)`（`--purge-retract`，默认 `1.0`）
  整块路径的最终停止点容易被腔内余压淤出堆料。设为正值时，在所有处理
  （含二次整形）结束后零流量再走一整圈，把该回抽量摊到整圈逐步完成——
  压力沿一圈释放，停止时已处于回抽状态，无静止回抽淤积。外墙原有的
  静止回抽/擦拭会被清零（由排压圈接管），回抽差额在圈尾补齐，E 收支
  与原文件一致。设 `0` 关闭。仅闭合回路生效。

- `最大悬垂角(度)`（`--max-overhang`，默认 `-90`=不生效）
  角度约定：`0` = 垂直墙，正 = 内缩（有支撑的正坡），负 = 外悬（悬垂）。
  悬垂角低于该阈值的弧长超过容差时，回路自动降级为 flat 模式：保留
  原始层高路径，只做流量渐变。降级数量在总结中报告。
  默认 `-90`：实际关闭检测（XY 插值已把下探路径拉向前层轮廓，悬垂
  自然有依托，无需降级）。若特定模型悬垂面质量异常，可设 `-15`~`-35`
  重新启用降级保护；设 `0` 或正值同样关闭。

- `末圈空转压平`（`--enable-flatten` / `--disable-flatten`，默认关）
  螺旋到顶补完互补圈后，零流量再走一圈把顶面熨平。默认关闭（空转圈
  会拖擦已挤出的表面，且排压圈已承担收尾功能）。

- `斜拼长度(mm)`（`--scarf-length`，默认 `3.2`）
  scarf 模式的斜面长度；螺旋模式不使用。

- `低于此墙长不斜拼(mm)`（`--min-wall-length`，默认 `4.0`）
  外墙周长高于该值（且高于 2 倍斜拼长度）时正常处理。

- `极限周长(mm)`（`--wall-limit`，默认 `1.0`）
  外墙周长低于该值时完全不做接缝处理（只二次整形）。

- `短墙兜底百分比`（`--short-fallback`，默认 `0.25`，上限 0.45）
  周长介于极限周长与"低于此墙长不斜拼"之间的短墙，不再直接跳过：
  按 周长 × 该百分比 作为等效斜拼长度参与处理（螺旋模式照常螺旋，
  scarf/flat 模式用缩短的斜面），小孔、小柱轮廓也能消接缝。

- `外墙首次流量缩放`（`--primary-flow`，默认 `0.96`）
  外墙的总流量缩放，略微减料给二次整形留余量。

- `接缝起点流量缩放`（`--seam-min-flow`，默认 `0.4`）
  仅 flat 模式使用：接缝两端的最低流量倍率。

### 二次整形

- `启用二次整形`（`--enable-secondary` / `--disable-secondary`，默认开）
  外墙完成后从偏移起点低流量重走一遍。起步/收尾自带流量渐变，防压力突起。

- `二次整形流量缩放`（`--secondary-flow`，默认 `0.08`）
- `整形速度倍率`（`--secondary-speed-scale`，默认 `1.0`=外墙原速）
- `整形接缝段速度倍率`（`--secondary-seam-speed-scale`，默认 `0.67`；仅 scarf/flat 模式使用）
- `起点绝对偏移(mm)`（`--secondary-start-offset`，默认 `2.6`）
- `短外墙相对偏移`（默认 `0.25`）
- `二次整形模式`（`--secondary-mode full_loop|local_window`）与`局部窗口长度(mm)`（`--secondary-window`，默认 `24`）

### 其他

- `螺旋速度倍率`（`--spiral-speed-scale`，默认 `0.66`）
  自适应速度：以每个回路外墙的原始打印速度为基准，按倍率缩放后用于
  螺旋圈、互补圈、空转圈、排压圈。Orca 按悬垂/小件自动调速的结果
  会被继承下来，不同区域各自跟随原速。默认 0.66 略降速，给亚层薄挤出
  留足熔融附着时间。

- `螺旋小角度降速`（`--enable-angle-speed` / `--disable-angle-speed`，默认关）与
  `角度速度曲线`（`--angle-speed-profile`，默认 `45:1.0,30:0.8,15:0.4,0:0.33`）
  按动态超分完成后的实际相邻子圈轨迹计算表面角度：`90` 度为垂直面，
  `0` 度为水平面。向外悬垂与向内堆叠均按水平位移绝对值计算，因此同角度
  使用相同倍率。默认曲线为 45 度及以上 1.0、30 度 0.8、15 度 0.4、
  0 度 0.33，档位之间线性插值。最终速度为
  `原外墙速度 × 螺旋速度倍率 × 角度倍率`；只修改螺旋、顶部互补圈和可选
  压平圈的进给速度，不修改轨迹坐标或挤出量。

- `采样精度(mm)`（`--sample-step`，默认 `0.2`）
  路径生成的细分步长，全局生效：长直边按该步长拆分，让螺旋 Z 爬升、
  XY 插值、流量渐变在长边中间也连续（否则只在原始顶点处取值）；
  G2/G3 圆弧也按该步长展开为直线段后正常处理（不再跳过圆弧回路）。
  越小越精细，G-code 行数越多。建议 0.1~0.5。注意它是插值/细分步长，不是几何偏差：直线细分不引入任何几何误差，圆弧展开在常见半径下 0.2mm 步长的弦差远小于 0.01mm。

- `空移速度(mm/s)`（`--travel-speed`，默认 `150`）
  插件插入的 G0 空移速度。

- `保留处理副本`（`--keep-copy` / `--no-keep-copy`，默认开）
  处理后把 G-code 存一份到插件目录 processed/ 下（保留最近 5 份），
  总结弹窗提供"查看处理后G-code"按钮直接定位文件。把该文件拖进
  OrcaSlicer 窗口即可用完整 3D 预览查看螺旋路径（Orca 支持直接打开
  .gcode 文件，预览里能看到 Z 爬升和流量着色）。

## 重要限制

1. 该插件通过 G-code 后处理工作，不会更新 OrcaSlicer 内部预览和打印时间估计。插件自己的总结弹窗给出加速度感知的耗时估计（梯形加减速 + 拐角限速 + 前瞻规划，跟随文件内 M204），与 Orca 原生估计口径接近（实测偏差 5%~10%，插件略偏保守）。
2. G2/G3 圆弧自动展开处理；展开后输出为直线段（文件稍大，等效关闭该回路的 Arc Fitting）。
3. 螺旋模式整圈都会把 Z 压到前一层与本层之间，要求外墙下方有实体支撑；悬空外墙（桥接段）不适用。
4. 螺旋模式下外墙在同层其他结构（内墙/填充）之前或之后打印均可，但**外墙先打（outer wall first）时喷嘴下探的碰撞风险最低**，建议在 Orca 中把墙打印顺序设为外墙优先。
5. 建议先在简单模型上测试，再逐步扩大使用范围。

## 验证与回归

`validate_gcode.py` 用临时副本处理和审计 G-code，不会改动原文件。推荐用当前 Orca 后处理命令先跑小到中等体积样本：

```powershell
python -B .\validate_gcode.py --root "C:\path\to\gcode" --recursive --max-size-mb 12 --processor-command-file .\orca_postprocess_command.txt --wall-thr 0.9 --surface-thr 0.6 --generated-thr 0.9 --per-layer 200 --per-layer-surface 900 --surface-step 1.1 --generated-step 1.0 --require-top-finish --max-generated-external 0 --max-generated-internal 0 --max-generated-unneeded-ironing 0 --max-transition-feed-ratio 4.0 --max-transition-flow-ratio 8.0
```

关键字段：

- `wall_bad` / `surface_bad` / `top_finish_missing` 必须为 0，分别表示外墙、裸露表面、最顶层熨烫/螺旋覆盖是否有漏区。
- `generated_external_bad` / `generated_internal_bad` / `generated_unneeded_ironing_bad` 必须为 0，分别表示外部非法生成路径、内部多余生成路径、不必要脚本熨烫。
- `pass_count_cv` 越低，说明各处被经过次数越一致；阈值可用 `--max-pass-cv` 控制，默认 0 表示只观察不判失败。不同模型自然差异较大，建议先看批量汇总里的最差文件再设阈值。
- `feed_ratio` / `flow_ratio` 和 `transition_feed_ratio` / `transition_flow_ratio` 用于观察生成路径内部、螺旋到脚本熨烫等衔接处的速度/流量突变。
- 已经带有 BOWP 标记的旧输出默认跳过；需要审计旧输出本身时加 `--audit-processed-self`。
- 批量模式末尾的 `batch_summary=begin/end` 汇总总失败数、漏覆盖数、生成路径安全计数、脚本熨烫线段数和最差指标文件。

## 回退方式

1. 在设置中关闭整个插件
2. 接缝模式降级：spiral -> scarf -> flat
3. 把二次整形模式改为 `局部窗口` 或关闭二次整形
